Modern Workforce IAM Structure

In today’s cyber world, IAM technology and market are shifting rapidly. A well-established IAM system will provide a much better foundation to grow its business, lowering cost and risk for the daily operations.

Customer IAM and Workforce IAM

In general, IAM can be categorized into two parts, Customer IAM (CIAM) and Workforce IAM (or Employee IAM). Due to the nature that identity targets, the core objects of an IAM system, are different between CIAM and Workforce IAM, the way they are treated or architected are different as well. For example, in Workforce IAM, employees are registered by HR, who can help to handle registration issues from various aspects; in CIAM, end users will usually register by themselves and this flow should be straightforward and intuitive.


Workforce IAM Requirements

There are various aspects we can talk about regarding Workforce IAM. Here are some of them.

Empower Work Efficiency

Work efficiency is a large topic. From a certain perspective, increasing work efficiency reduces operational cost while maintaining productivity and hence increased profit. Empowering a SSO mechanism across different platforms ranging from internal in-house application to external partners service application will greatly reduce the application access friction and workforce disconnectivity. Self-service portal for employees will enhance user experience and reduce IT costs as well.

Authorization with Flexible Policy

Enterprise is never a standalone entity, especially in this globally connected informational world. We are all seeing an increasing complexity among an enterprise application system, both internally and externally. An IAM system that provides a flexible Authentication and Authorization enforcement mechanism which handles different workflows at different granularity will no doubt enhance the user experience. For example, an employee accessing an internal application which doesn’t interact with sensitive information doesn’t need to enforce an MFA while an external partner accessing an internal application from the public internet will definitely need to have an MFA.

Legacy App Handling

Legacy applications could be painful to handle, especially for those enterprises with some history. Due to the incompatibility and poor support with modern technology, its integration could be challenging. A good IAM platform should provide flexible ways to integrate with legacy apps.


Continuous Monitoring

Continuous Monitoring is a continuous journey. A good IAM system should provide strong logging and monitoring mechanisms. This could be due to troubleshooting demand, regulatory compliance needs, or any other reasons.

Components Architecture

Here is a general architecture using Ping Identity for Workforce Management.

Workforce IAM Components Diagram

Use Case Scenario

  • Centrified Authentication and Authorization Provider

  • SSO across internal and external platforms

  • Gateway proxy protecting legacy apps

  • Adaptive MFA for sensitive accesses

  • Self-service for reduced IT costs

  • Continuous monitoring with external Saas platform


PingFederate here is acting as a central authentication and authorization management platform. An enterprise AD is connected to it for user profile provisioning. SSO is fulfilled via such modern protocols as OAuth, OIDC and SAML. With Auth Provider’s built-in authorization policy management, we can implement comprehensive authorization workflows to adapt to different needs.


Nevertheless, PingAccess can act as a gateway proxy as a policy enforcement point to protect legacy applications while leveraging PingFederate as policy decision point. To further enhance security, PingID cloud service is used with PingFederate to provide MFA capability. Monitoring is log forwarding from all sources to external aggregating Saas platforms such as Splunk.


This architecture is leveraging Ping Identity products, but can be extended to other IAM providers as well.
